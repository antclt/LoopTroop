# Code of Conduct

LoopTroop is an open-source project for local AI coding orchestration. We want the project space to be useful, direct, and respectful for people reporting bugs, discussing workflow ideas, reviewing code, and improving documentation.

## Expected behavior

- Be respectful and constructive in issues, pull requests, reviews, and discussions.
- Assume good intent, especially when someone is reporting rough edges in early alpha software.
- Keep feedback focused on the work: behavior, code, docs, tests, user experience, and safety.
- Be patient with maintainers and contributors. Clear reproduction steps and concrete examples help more than urgency alone.
- Respect privacy. Do not post secrets, private repository content, API keys, tokens, or logs containing sensitive data.

## Unacceptable behavior

- Harassment, insults, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeatedly derailing issues or pull requests away from the project topic.
- Abusive, spammy, or intentionally disruptive behavior.

## Reporting problems

If you see behavior that violates this code of conduct, contact the maintainers at `contact@looptroop.ovh`.

Please include enough context to understand what happened, such as links to the issue, pull request, comment, or discussion. Maintainers will review reports in good faith and may remove content, lock conversations, or block participants when needed to keep the project healthy.
